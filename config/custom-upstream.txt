# Резолвинг отдельных доменов через свой апстрим-DNS (policy.FORWARD в Knot Resolver)
#
# Полезно, когда дефолтный резолвер AntiZapret отдаёт для домена IP из
# заблокированной/недоступной подсети (гео-DNS), а другой резолвер возвращает рабочий.
# Домен остаётся на ПРЯМОМ маршруте (НЕ через VPN) — меняется только источник ответа.
#
# Формат записи: <домен> <резолвер[,резолвер2,...]>
# Где:
#   домен    - доменное имя в ASCII/Punycode (домен и все его поддомены)
#   резолвер - IPv4-адрес DNS-сервера (несколько — через запятую или пробел)
#
# Примеры записи:
#   example.com            1.1.1.1
#   sub.example.com        1.1.1.1,9.9.9.10
#
# Приоритет: custom-bind.txt > custom-upstream.txt > маршрутизация AntiZapret
# Строки начинающиеся с # это комментарии и они не обрабатываются
#
# Пример: чинит «вечную крутилку» App Store на части RU-аплинков —
# amp-api резолвится через CF/Quad9 в рабочий Fastly-префикс (151.101.x)
# вместо гео-DNS, который отдаёт заблэкхоленный 146.75.119.x.
# Раскомментируйте, если не грузится App Store:
#amp-api.apps.apple.com        1.1.1.1,1.0.0.1,9.9.9.10,149.112.112.10
#amp-api-edge.apps.apple.com   1.1.1.1,1.0.0.1,9.9.9.10,149.112.112.10
